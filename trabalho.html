Lógica de Programação

Introdução à Lógica de Programação

A lógica de programação é o conjunto de regras e técnicas que os programadores utilizam para projetar e desenvolver programas de computador. É a habilidade de pensar de forma lógica e estruturada, decompondo um problema complexo em etapas mais simples.

Em outras palavras, é a capacidade de organizar informações e comandos de forma que um computador possa seguir as instruções e executar as tarefas desejadas. A lógica de programação é essencial para desenvolver qualquer programa ou software, pois define como o computador deve processar as informações e realizar as ações.

Por que é importante?

A lógica de programação é fundamental para:

  • Resolver problemas: Ajuda a encontrar a melhor maneira de abordar e solucionar problemas complexos, decompondo-os em etapas mais simples.

  • Pensar de forma estruturada: Estimula o pensamento lógico e a organização das ideias.

  • Aprender linguagens de programação: A lógica de programação serve como base para aprender qualquer linguagem de programação, facilitando o processo de escrita e entendimento do código.

  • Desenvolver habilidades de programação: Aprimora a capacidade de criar soluções eficientes e bem estruturadas.

Sempre que optamos por realizar alguma tarefa em nossa rotina, acabamos seguindo uma ordem lógica.

Na maioria das vezes, fazemos isso de forma tão automática que nem percebemos. No entanto, se refletirmos um pouco, conseguimos identificar algumas etapas que nos conduziram ao resultado final.

Pense em uma situação em que você precisa fazer um bolo. Esses possivelmente serão os passos:

  1. Selecionar os ingredientes da receita;

  2. Selecionar tigela;

  3. Colocar farinha, de acordo com a medida;

  4. Selecionar ovos;

  5. Colocar manteiga e açúcar a gosto;

  6. Colocar leite;

  7. Misturar todos os ingredientes na tigela;

  8. Despejar a massa na forma;

  9. Levar ao forno;

  10. Levar ao forno;

  11. Retirar do forno;

  12. Servir o bolo.

Estruturas de controle

As estruturas de controle condicional Se/Senao (if/else) permitem que um programa execute diferentes blocos de código dependendo de uma condição. Se a condição for verdadeira, o bloco associado ao Se é executado; caso contrário, o bloco associado ao Senao (se existir) é executado.

- Se (if):

A estrutura Se inicia a condição a ser avaliada. A condição é uma expressão booleana que resulta em verdadeiro ou falso.

- Condição:A condição é avaliada. Se for verdadeiro, o bloco de código dentro do Se é executado. - Senao (else):

A estrutura Senao (opcional) é executada quando a condição do Se é falsa. Se não houver Senao, o programa simplesmente ignora o bloco Se e continua com a execução.

- Bloco de código:

O bloco de código dentro do Se e/ou Senao é a sequência de instruções que são executadas dependendo do resultado da condição.

Exemplo com (Se/Senao):

            se (idade >= 18)
               escreval("Você é maior de idade.");
          senao
               escreval("Você é menor de idade.");

Variáveis e Constantes

Em estruturas de dados básicas, variáveis são espaços na memória que armazenam valores que podem ser alterados durante a execução de um programa, enquanto constantes são valores fixos e não podem ser modificados.

Variáveis

Definição:

Uma variável é um espaço reservado na memória para armazenar um valor que pode mudar durante a execução de um programa.

Propriedades:
  • Nome: Cada variável tem um nome único para identificar o seu espaço na memória.

  • Tipo: O tipo de dado define o tipo de valor que a constante pode armazenar.

  • Valor: O valor da constante é fixo e não pode ser alterado.

Os principais dado são:

  • Inteiros: 1, 2, -3
  • Decimais: 3.14, -0.01
  • Texto (String): "Olá mundo"
  • Lógico (Booleano): true, false

O que é Algoritimo?

Em programação, um algoritmo é um conjunto de instruções que descreve uma sequência de operações para resolver um problema específico ou realizar uma tarefa computacional. Ele é como uma receita de bolo para a máquina, indicando passo a passo como transformar uma entrada em uma saída desejada.

Um algoritmo pode ser representado por:

Pseudocódigo

  • Pseudocódigo não é uma linguagem de programação, mas sim uma ferramenta para descrever os passos de um algoritmo de forma clara e organizada.

  • É usado para planejar, projetar e entender a lógica de um programa antes de ser escrito em uma linguagem de programação específica.

  • Utiliza uma linguagem natural, como português ou inglês, com algumas convenções para representar estruturas de controle (como loops e condicionais) e operações.

Fluxograma

É uma representação visual e gráfica de um algoritmo ou processo, que descreve a sequência de passos, as decisões e o fluxo de dados. É uma ferramenta utilizada para planejar, entender e comunicar a lógica de um programa antes mesmo de se escrever o código.

  • Ajuda a organizar e estruturar a lógica do algoritmo antes de começar a codificar.

  • Facilita a compreensão do algoritmo tanto para o desenvolvedor quanto para outras pessoas envolvidas no projeto.

  • Serve como uma ferramenta de comunicação visual para explicar o processo de forma clara e concisa.

  • Permite identificar possíveis problemas ou erros no algoritmo antes de executá-lo.

  • Pode ser usado como uma forma de documentar o algoritmo, tornando-o mais fácil de entender e manter no futuro.
